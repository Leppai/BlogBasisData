SQL, DDL, DML, DCL, dan TCL

Apa itu SQL, DDL, DML, DCL, dan TCL?

1. SQL (Structured Query Language)

SQL adalah bahasa standar untuk mengelola dan mengakses database relasional. Dengan SQL, kamu bisa membuat tabel, menyimpan data, mengubah data, menghapus data, hingga mengatur hak akses pengguna. SQL terbagi menjadi beberapa kategori perintah, yaitu DDL, DML, DCL, dan TCL.

2. DDL (Data Definition Language)

DDL digunakan untuk membuat, mengubah, dan menghapus struktur database. Semua operasi DDL bersifat auto-commit, artinya perubahan langsung tersimpan.

Contoh perintah DDL:

  1. CREATE â†’ membuat tabel atau objek baru
  2. ALTER â†’ mengubah struktur tabel
  3. DROP â†’ menghapus tabel
  4. TRUNCATE â†’ menghapus seluruh data dalam tabel tanpa menghapus strukturnya

3. DML (Data Manipulation Language)

DML digunakan untuk mengolah data di dalam tabel. Perubahan DML tidak langsung disimpan sebelum dilakukan COMMIT.

Contoh perintah DML:

  1. SELECT â†’ mengambil data
  2. INSERT â†’ menambah data
  3. UPDATE â†’ mengubah data
  4. DELETE â†’ menghapus data

4. DCL (Data Control Language)

DCL digunakan untuk mengontrol hak akses dan izin pengguna terhadap data dalam database.

Contoh perintah DCL:

  1. GRANT â†’ memberi hak akses
  2. REVOKE â†’ mencabut hak akses

5. TCL (Transaction Control Language)

TCL digunakan untuk mengelola transaksi agar perubahan data tetap aman dan konsisten.

Contoh perintah TCL:

  1. SAVEPOINT â†’ membuat titik penyimpanan agar bisa rollback sebagian
  2. ROLLBACK â†’ membatalkan perubahan sebelum commit
  3. COMMIT â†’ menyimpan perubahan permanen
